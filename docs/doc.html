Literals, identifiers, variables

An identifier can be in format

tbl_name | db_name.tbl_name | db_name.tbl_name.col_name

A literal is

'string' | 123 | 123.45 | TRUE | FALSE;

A variable is

@variable_name;

    Aggregation


    Binary expressions can be used with the following functions:
    • AVG
    • COUNT
    • MAX
    • MIN
    • SUM

    Using binary expression with aggregation function SELECT <func>(<identifier>)+1;

    You cannot nest aggregation functions.


    COUNT

    The count function returns the number of rows that matches the query. It can be used with the asterisk (*) to count all rows.

    You can pass a column name to count the number of rows that are not null in that column.

    SELECT COUNT(*) FROM <identifier>; SELECT COUNT(<identifier>) FROM <identifier>;

    AVG

    The avg function returns the average of the values in the column.

    SELECT AVG(<identifier>) FROM <identifier>;

    MAX

    The max function returns the maximum value in the column.

    SELECT MAX(<identifier>) FROM <identifier>;

    MIN

    The min function returns the minimum value in the column.

    SELECT MIN(<identifier>) FROM <identifier>;

    SUM

    The sum function returns the sum of the values in the column.

    SELECT SUM(<identifier>) FROM <identifier>;

    Aliases

    You can use aliases within a select list. With column specifications, binary expressions, or aggregate functions.


    SELECT <column spec>|<aggregate function>|<binary expression> AS <identifier> FROM <identifier>;

    From Clause

    In the from clause, you can use the following syntax to join tables or not.


    FROM <identifier> | <identifier> AS <identifier> | <identifier> <identifier>; FROM tbl1 | tbl1 AS a | tbl1 a; FROM <tables> FROM t1 a, t2 b;

    Where Clause

    The where clause is used to filter rows that match a condition.
